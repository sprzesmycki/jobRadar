# Code Review

Use this skill when reviewing a pull request or a local diff.

## Steps

1. Read the full diff — understand what changed and why.
2. Flag correctness bugs first: off-by-one, null handling, error paths.
3. Note reuse and simplification opportunities, but keep them separate from bugs.
4. Report findings most-severe first; skip nitpicks the team's linter already covers.
