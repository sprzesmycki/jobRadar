## Team Conventions

- Run the linter and the test suite before every commit; do not skip either side.
- Keep changes surgical — touch only what the task requires, match existing style.
- Prefer small, reviewable pull requests over large batched ones.
- Write commit messages in the Conventional Commits format.

These are example team rules bundled by the shared AI toolkit. The installer
splices this block into a consumer's `CLAUDE.md` between sentinel markers.
